# Infection Control Command Center

Multi-page hospital-acquired infection (HAI) surveillance dashboard built on **real, public CDC/CMS data** — no synthetic data anywhere in the core analysis (a supplementary control-chart demo page simulates a monthly series purely for technique demonstration, clearly labeled).

## Data sources

- **CMS Provider Data Catalog**, dataset [`77hc-ibv8`](https://data.cms.gov/provider-data/dataset/77hc-ibv8) — *Healthcare Associated Infections - Hospital* (measures collected by CDC's National Healthcare Safety Network)
- **CMS Provider Data Catalog**, dataset [`xubh-q36u`](https://data.cms.gov/provider-data/dataset/xubh-q36u) — *Hospital General Information* (hospital type, ownership, ER services, CMS star rating — real features for the predictive model)
- 3,119 hospitals, 6 infection types (CLABSI, CAUTI, SSI-Colon, SSI-Hysterectomy, MRSA Bacteremia, C. difficile), most recent 12-month reporting period (07/01/2024–06/30/2025)

## Dashboard pages

1. **Executive Summary** — KPIs, headline findings, methodology summary, downloadable HTML report.
2. **Facility Benchmarking** — geographic choropleth map (state-level volume-weighted SIR) + funnel-plot control charts (facility SIR vs. predicted infection volume, with Poisson-based statistical control bands).
3. **Statistical Testing** — exact Poisson test p-value per facility-measure, with Benjamini-Hochberg FDR correction across all ~11,300 simultaneous tests, interactive alpha threshold, p-value-vs-q-value diagnostic plot.
4. **Predictive Model** — logistic regression + gradient boosting predicting which facility-measures get flagged worse-than-benchmark, trained on real hospital metadata (type/ownership/ER services/star rating) + infection type. ROC-AUC, PR-AUC, confusion matrix, feature importance — explicitly handling a ~1.7% positive-rate class imbalance.
5. **Control Chart Demo (synthetic)** — CUSUM/EWMA charts on a simulated monthly series anchored to one real facility's real SIR, since CMS doesn't publish per-facility monthly history. Clearly labeled synthetic throughout.

## Why a funnel plot instead of a time-series control chart (main benchmarking page)

CMS publishes a single most-recent 12-month rolling period per snapshot — no per-facility monthly history is public. Rather than fabricate a trend line for real, named hospitals, the main benchmarking page uses **funnel plots**: SIR plotted against predicted infection volume, with statistical control bands (±2σ/3σ, derived from Poisson variance `1/predicted_infections`) — the standard technique real infection-control epidemiologists use to separate true outliers from small-sample noise.

## Key findings

- 189 facility-measures are flagged worse than the national benchmark by CMS's CI-based criteria; 16 facilities are flagged on **2+ infection types simultaneously**.
- Running ~11,300 simultaneous statistical tests inflates false positives: an uncorrected threshold flags **~28% of all tests**, while FDR correction narrows genuine worse-than-benchmark outliers to **58**.
- A real-data-only predictive model (hospital type/ownership/ER services/star rating + infection type) reaches **ROC-AUC ~0.80** at predicting which facility-measures get flagged, with CMS star rating showing the expected inverse relationship (higher rating → lower odds of being flagged).

## Run it

```bash
git clone <this-repo-url>
cd infection-control-surveillance
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

python notebooks/00_download_data.py        # pulls both real CMS datasets fresh
python notebooks/01_data_prep.py
python notebooks/03_stats_and_features.py
python notebooks/04_predictive_model.py
streamlit run dashboard/app.py
```

Data files (`data/*.csv`, `*.parquet`, `*.duckdb`, `*.joblib`) are gitignored and regenerated by the scripts above rather than committed, so the repo stays small and always reflects the live CMS dataset rather than a stale snapshot.

## Design system

`common/theme.py` — a palette validated with a six-check color-accessibility validator (lightness band, chroma floor, CVD-safe separation, contrast vs. surface) rather than eyeballed colors, plus a consistent KPI card style and Plotly chart template. (This module is shared with the companion [Surgical Complications & Mortality Benchmarking](https://github.com/DevSanjayShah05/surgical-complications-benchmarking) project so the two dashboards read as one coherent design system.)
